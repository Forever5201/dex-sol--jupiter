# 🎯 TP钱包私钥直接使用指南

## ✅ 好消息：TP钱包私钥可以直接使用！

**无需转换！** TP钱包导出的 Base58 私钥可以直接填入 `.env` 文件。

---

## 🚀 三步完成

### 步骤 1：从 TP钱包 导出私钥

1. 打开 **TP钱包（TokenPocket）**
2. 进入 **钱包设置** → **导出私钥**
3. 复制显示的 **Base58 私钥**（通常是一串 87-88 个字符的字符串）

### 步骤 2：验证私钥

```powershell
# 验证 TP钱包导出的私钥（替换为您的私钥）
pnpm tsx scripts/verify-tp-key.ts 你的Base58私钥
```

**示例：**
```powershell
pnpm tsx scripts/verify-tp-key.ts 5Kb8Kk8Lf9ioNdXL...（您的私钥）
```

脚本会显示：
- ✅ 私钥验证结果
- 💰 钱包地址
- 💰 余额信息

### 步骤 3：填入 .env 文件

验证通过后，在项目根目录创建或编辑 `.env` 文件：

```bash
# .env 文件
SOLANA_PRIVATE_KEY=你的Base58私钥（从TP钱包复制的）
```

**就这么简单！** ✅

## 💡 为什么不需要转换？

1. **TP钱包导出的是 Base58 格式**
   - 这正是系统需要的格式
   - 无需任何转换

2. **系统自动识别**
   - `KeypairManager` 会自动识别 Base58 格式
   - 支持 32 字节或 64 字节私钥

3. **直接使用**
   - 从钱包导出 → 验证 → 填入 .env → 完成！

## ❓ 常见问题

### Q1: TP钱包导出的私钥格式是什么？

**A:** TP钱包导出的是 **Base58 格式**的私钥，通常是一串 87-88 个字符的字符串，例如：
```
5Kb8Kk8Lf9ioNdXL...（87-88个字符）
```

### Q2: 可以直接使用吗？

**A:** 是的！**完全可以直接使用**，无需任何转换！

### Q3: 如果验证失败怎么办？

**可能原因：**
- 私钥复制不完整
- 包含了多余的空格或换行
- 私钥格式不正确

**解决方案：**
1. 重新从 TP钱包 导出私钥
2. 确保完整复制（不要遗漏任何字符）
3. 检查是否有空格或换行

### Q4: 其他钱包（如 Phantom）也可以吗？

**A:** 是的！**任何导出 Base58 格式私钥的钱包都可以直接使用**：
- ✅ TP钱包（TokenPocket）
- ✅ Phantom 钱包（如果导出 Base58 格式）
- ✅ Solflare 钱包
- ✅ 其他支持 Base58 私钥的钱包
